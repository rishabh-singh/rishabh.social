    Description: A customer had his website running over Wordpress. The website was hosted on EC2 on a LAMP environment. The website and all it's contents were all on the same machine, with the database being on RDS. The idea was to seperate each component of the website, to make it stateless and use Docker containers to run the Wordpress engine.

    Detailed Implementation is here.

  • Automation of Amazon Machine Image(AMI) creation using AWS SDK for Java.

    • Project Scenario: Automation of AMI creation for multiple customers

    Description: Being a managed service provider, my first comapny had several customers which had their workload running on AWS Cloud. The customers had several VM, for which backup was required. Creating AMI was one major backup strategy which we depended on. The creation process was manual. Retention was also to be ensured as many copies of AMI would incur cost. There was no tracking availabe and thus when the number of clients and their respective VMs increased, it was getting difficult to manage this backup process. This solution was fully automated using AWS SDK for Java. Mailing feature was introduced on successful and failed backups for efficient tracking.

    Detailed Implementation is here.
